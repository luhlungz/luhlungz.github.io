# Microsoft 365 Admin Case Studies

## Overview
This repository contains anonymised Microsoft 365 support case studies that demonstrate practical troubleshooting, investigation and documentation skills.

## Case Studies

### 1. Suspicious Email Investigation
**Scenario:** A user reports a suspicious email.

**Process:**
- Confirm user, mailbox, sender, subject and time received.
- Check Exchange Online message trace.
- Review delivery status: delivered, blocked, quarantined or redirected.
- Check for suspicious inbox rules.
- Review sign-in activity and MFA registration if compromise is suspected.
- Take action and document the outcome.

**Skills:** Exchange Online, message trace, MFA checks, security awareness, documentation.

### 2. Shared Mailbox Access Issue
**Scenario:** A user cannot access or send from a shared mailbox.

**Process:**
- Confirm whether the issue is Full Access, Send As, Send on Behalf or Outlook visibility.
- Check permissions in Exchange Admin Center.
- Apply correct permission if approved.
- Ask user to test Outlook on the web if Outlook desktop caching is suspected.
- Document resolution.

**Skills:** Shared mailbox support, Exchange permissions, user communication.

### 3. VPN / NetExtender Access Support
**Scenario:** A remote user cannot connect using VPN / SonicWall NetExtender.

**Process:**
- Confirm error message, device and network status.
- Confirm group membership such as SSLVPN / SSL-VPN/NetExtender access group.
- Confirm MFA/credential behaviour where applicable.
- Test client configuration.
- Escalate if firewall policy or wider issue is suspected.

**Skills:** VPN support, access group checks, troubleshooting, escalation.
