Microsoft 365 Admin Case Studies

Sanitised support examples covering suspicious email checks, mailbox access, MFA checks and VPN support workflow.

Case study 1: Suspicious email investigation

Scenario: A user reports a suspicious email or unusual mailbox behaviour.

Process

  • Confirm sender, subject, time received and affected mailbox.
  • Use Exchange message trace to verify whether the message was delivered, blocked, quarantined or redirected.
  • Check for suspicious inbox rules or forwarding.
  • Review sign-in activity and MFA registration status if account compromise is suspected.
  • Take action: block sender/domain, remove malicious rules, reset password, force re-authentication or escalate where needed.
  • Document the investigation and final outcome in the ticket.
Skills demonstrated: Microsoft 365 support, Exchange Online troubleshooting, message trace, MFA checks, security awareness and documentation.

Case study 2: Shared mailbox access issue

Scenario: A user cannot access or send from a shared mailbox.

Process

  • Confirm whether the issue is access, Send As, Send on Behalf, visibility in Outlook or caching delay.
  • Check Full Access and Send As permissions in the Exchange Admin Center.
  • Confirm the affected user and mailbox are correct.
  • Ask the user to restart Outlook or test Outlook on the web where needed.
  • Document the permission changes and expected propagation time.

Case study 3: VPN/NetExtender access support

Scenario: A user cannot connect with SonicWall NetExtender or remote VPN/NetExtender access.

Process

  • Confirm the user, device, error message and network status.
  • Check whether the user is in the correct SSLVPN / SSL-VPN/NetExtender access group.
  • Confirm credentials and MFA requirements where applicable.
  • Test client installation, server address and internet connectivity.
  • Escalate if the issue points to firewall policy, licensing or wider outage.

Portfolio note

These examples are intentionally anonymised. No client names, mailbox addresses, ticket numbers or private screenshots should be published.
